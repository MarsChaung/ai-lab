CPU 需求復活的背後

AI 服務商對 CPU 需求極強，甚至消化了舊規格庫存。當模型走向「每天被數十億次使用」的推論階段，大量 CPU 成為處理搜尋、程式碼與系統串接的低成本首選。

代工業務的雙面刃

雖然營收成長 16%，但營業虧損仍高達 24.37 億美元。市場關注焦點在於：AI CPU 產生的利潤能否支撐龐大的製造端投資？

Meta 與 AWS 協議：多晶片策略的轉折

Meta 與 Amazon 達成的數十億美元協議，重點在於導入「數千萬個」Graviton 核心來支撐下一代 Agentic AI 工作負載。

  • ✔
    降低成本 自研晶片節省的成本可直接回饋予 Meta，解決 GPU 過於昂貴的問題。
  • ✔
    多步驟協調 Graviton5 的快取提升 5 倍，大幅降低核心間延遲，適合 AI 推論。
  • ✔
    分散供應風險 Meta 採取 Portfolio Approach，不將所有雞蛋放在 Nvidia 單一籃子裡。
  • ✔
    推論工作專門化 程式碼生成、即時推理、搜尋等任務，正從 GPU 轉移至高效能 CPU。
